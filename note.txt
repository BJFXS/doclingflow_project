1. PDF 是主要瓶颈
docx / pptx / html 基本都在 2–3 秒左右完成，而 PDF 经常需要几十秒到几百秒。

2. 长 PDF / 图片型 PDF / 图表多 PDF 容易 timeout
这说明转换器不是完全不能处理 PDF，而是遇到某些 PDF 时无法稳定完成
    - 在 file_analyzer.py 里先分析 PDF：页数，文件大小，是否图片很多，是否扫描件，是否双栏，是否图表密集
    - 然后在 strategy_selector.py 里决定策略：
        短 PDF → 正常转换
        长 PDF → 分页/分块转换
        图片型 PDF → 开 OCR 或走备用 converter
        图表多 PDF → 降低处理精度，或者跳过图片深度解析

3. 资源占用很高，一个文件失败影响整个批量任务
如果用户一键转换整个目录，不应该因为一个 PDF 失败导致整个任务卡住。
    - 一个文件一个 subprocess，失败后记录报告，不中断整体任务。

4. 图片处理不完整
    - 做一个 image_handler.py，是否用视觉模型可以做成可选参数，普通用户默认 extract，需要 LLM/RAG 高质量理解时开启 vlm-caption
        1. 抽取图片到 output/images/
        2. 在 Markdown 中插入图片链接
        3. 给图片生成占位说明
        4. 可选：调用视觉模型生成 image caption

5. 输出 Markdown 可能过大，不适合直接给 LLM
有些 PDF 输出的 Markdown 很大，例如：long_02.pdf → 645,092 characters
这类输出即使转换成功，也不能直接塞给 LLM。后续还需要 chunking、清洗、去重、结构化。

6.部分结构保留不理想
有些 DOCX / HTML 的 heading_count 是 0，说明标题层级可能没有被很好地转换成 Markdown heading。
    - 识别大写短行作为标题、识别编号标题，例如 1. Introduction、识别 PPTX slide title、识别 HTML h1/h2/h3、识别 DOCX paragraph style
    - 然后补成：## Section Title

1. 长 PDF / 图片型 PDF / 图表多 PDF 容易 timeout
问题：
    遇到复杂 PDF 时耗时很高，甚至无法稳定完成转换
    用户一键处理整个目录时，容易被少数重型文件拖慢
解决方法：
    - 在 file_analyzer.py 中预分析 PDF：页数、文件大小、图片比例、是否扫描件、是否双栏、是否图表密集
    - 在 strategy_selector.py 中按类型分流：
        短 PDF → 正常转换
        长 PDF → 分页 / 分块转换
        图片型 PDF → 开 OCR 或切换备用 converter
        图表多 PDF → 降低处理精度，跳过图片深度解析


2. 资源占用很高，一个文件失败影响整个批量任务
问题：
    某个文件超时或崩溃，可能拖住整个批处理流程
    大文件容易占满 CPU / 内存
解决方法：
    - 一个文件一个 subprocess
    - 为每个任务单独设置 timeout、内存限制、失败重试次数
    - 文件失败后只记录日志和报告，不中断其他文件

3. 图片处理不完整
问题：
    图片常常只被占位，不能真正成为可用内容
    图表也常被当作普通图片处理
解决方法：
    - 做一个 image_handler.py，将图片处理单独模块化
    - 默认模式：只抽取图片并插入 Markdown 链接
    - 增强模式：补充占位说明
    - 可选模式：调用视觉模型生成 image caption，提升 LLM/RAG 可用性

4. 部分结构保留不理想
问题：
    有些 DOCX / HTML / PPTX 的标题没有被转成 Markdown heading
    列表、表格、引用块虽然保留了内容，但结构不够清晰
解决方法：
    - 做一个 structure_repair.py，在转换后修复结构
    - 识别短标题、大写标题、编号标题、HTML h1/h2/h3、DOCX paragraph style、PPTX slide title
    - 自动补成 # / ## / ###
    - 对列表、表格、引用块做格式标准化，提高 preview 和 chunking 效果

5. 复杂公式 / 双栏论文 / 图表语义保留不足
问题：
    简单公式还能转，复杂公式和数学符号容易丢失或排版错乱
    双栏论文可能存在顺序问题
    图表只保留为图片，语义信息不足
解决方法：  
    - 做一个 special_block_handler.py 或 semantic_enhancer.py
